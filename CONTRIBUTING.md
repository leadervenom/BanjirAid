# Contributing to BanjirAid

## Branch rules
- Do NOT commit directly to `main`
- Work on `dev` or `feature/*` branches
- Open a Pull Request (PR) into `dev`
- Only merge `dev` â†’ `main` when stable (demo-ready)

## Branch naming
- feature/flutter-intake
- feature/python-triage
- feature/dashboard-queue
- feature/firebase-rules
- fix/<short-bug>

## Commit messages
Use: type: message
Examples:
- feat: citizen request form
- feat: triage endpoint
- fix: ticket status update bug
- docs: update prompt log

## Definition of Done (DoD)
- Code runs locally
- No secrets/API keys committed
- Basic test or manual check done
- PR description explains what changed
